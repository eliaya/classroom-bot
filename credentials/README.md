# Google Classroom credentials

Place OAuth files here (mounted into containers at `/app/credentials/`):

| File | Purpose |
|------|---------|
| `client_secret.json` | Download from Google Cloud Console (OAuth 2.0 Client ID) |
| `token.json` | Generated by the setup script (do not edit by hand) |

**Do not commit these files to Git.**

## First-time setup (on the host, not inside Docker)

```bash
# From project root (creates .venv and installs deps automatically)
./scripts/setup-google-auth.sh
```

Or manually with the project venv:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python src/scripts/setup_google_auth.py
```

A browser window opens for Google sign-in. The script writes `credentials/token.json`.

## After upgrading scopes or token errors

If the web UI shows *Google credentials missing or invalid* or *missing required Classroom scopes*, re-authorize:

```bash
./scripts/setup-google-auth.sh
docker compose restart api bot
```

## Verify

```bash
curl -s http://127.0.0.1:8000/api/status | python -m json.tool
```

`google_credentials` should be `"valid"`. The `google.missing_scopes` array should be empty.
